## 2024-05-18 - Missing Constraint on Tesoreria Account
**Vulnerability:** The `comprar_bloque` (now `comprar_bloques`) instruction in `programs/casino_solana/src/lib.rs` receives an account named `tesoreria` via `AccountInfo<'info>`. However, there is no validation on the `tesoreria` account to ensure it matches the actual designated treasury wallet.
**Learning:** Any user can pass their own wallet as the `tesoreria` account. Since the `tesoreria` account is not constrained, the Anchor program will successfully transfer the user's SOL (or another token) to the provided `tesoreria` account, which could simply be their own wallet or another attacker-controlled wallet. This allows users to "buy" blocks without effectively paying the protocol, essentially bypassing the game's economy. This is a critical security vulnerability in an on-chain economy.
**Prevention:** Always hardcode the expected treasury address using a constant (e.g., `pub const TESORERIA_WALLET: Pubkey = pubkey!("...");`) and use Anchor constraints (e.g., `#[account(mut, address = TESORERIA_WALLET)]`) to enforce that the provided account matches the expected address.
